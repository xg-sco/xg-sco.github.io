可以考虑找个 IT 技术支持，俗称网管，无非就是，装电脑，装系统，调网络，打印机。能基本操作以及面向百度解决问题，可以先混口饭吃。入了门，在研发部里多观察，最好找岗位的时候就强调未来想转运维（你说了会点 linux ），碰到好的 leader 愿意带你，你也可以参与到项目上线的流程。饭碗有了，身边环境也是技术性的，可以尝试转运维开发，接触开发知识，让你的运维更牛逼。学的好，最后再转开发也不是问题。

@peterli427 #59 同意楼上这位老兄的观点，我大学刚毕业的时候水平和你描述的这个状况差不多。本科心理学，后来进了一个心理学相关公司。做的就是你说的 IT 技术支持，收入的话，在 IT 这个行当里算是底层了，但还是比我本来的专业平均收入水平高，这个岗位也干得还算开心。

我不是啥都会点，啥都不精通么？巧了这岗位需求也是这，整个公司一共就三四十来个人，我就是这里的电脑水平天花板了。大家也不知道我进度，该划水划水，到点下班。做的工作大部分都是自己感兴趣的，举个例子：比如给办公室组网。一共两层楼，要求在同一内网，网内要部署一个 NAS ，让大家有一个共同的网盘。我纯粹边学边搭，反正你要说难吧，三四十号人，一共也就一百来个电子设备。也遇不到太复杂的问题，自学都能慢慢搞定。

公司小，老板也比较信任我，在这待了两年，顺着公司需求折腾了一堆东西，路由器、NAS 、磁带机备份、直播推流、基础的布光置景、拍摄宣传视频、基础的网站服务器运维之类的。

上面这些东西有不少老哥是自己感兴趣业余时间搞，我这直接上班时间搞。虽然捣鼓磁带机、捣鼓新相机做直播、买一堆 18T 硬盘组 raid 阵列。这些在技术大佬面前都是很入门的东西。但是没事啊我菜，我全都不会，让我自己花钱，A7M4 相机+镜头，这一套价格我肯定舍不得买。但是花公司钱，每个都能捣鼓一点，很开心，关键是搞出来之后公司里还有人用。比如 ChatGPT 4 出来那会儿，我自己摸索着，跟着教程在公司内网部署了一个套壳网站，拿公司的钱买了 token ，开放给大家用，大家都很开心，说这个好。虽然东西没啥技术难度，但是有人用就还是蛮有成就感的。

上面说的是优点：找这种要求不高的工作你干得开心，可以折腾自己喜欢的东西。
接着说下缺点：不太能指望这类公司有多少上升空间，以及没人带全靠自学，这样对比那种进了大公司，有靠谱前辈带着走的，进度会慢很多。不过反正按我毕业那会儿的水准，这种岗位多半也碰不着。

我待了两三年，目前找准方向打算往更专业的运维这条路再走走看，也就是从桌面运维转向服务器运维这类。接下来能找到合适岗位就跳槽，找不着就利用闲暇时间做做副业。反正不涉及开发，工作并不卷。

在这里大家的建议其实都是小马过河，都是本着自己的经验，各说各的。要按我这两年的经验就是：既然你对 IT 有兴趣，那就先顺从自己的兴趣，毕竟做感兴趣的事情是最省力的。大学刚毕业，如果自己目标不是很清晰，也没有高人指路的话，啥都试试看嘛，大不了走走弯路。

最近看了不少实习生的简历了，大部分项目经历写的都不怎么样，基本上一眼就知道是个什么水平的项目

都是 XXX 管理系统或者教程上的项目，拉不开差距

而且写的不怎么样，项目功能一句话带过，工作内容写成了技术选型
例如我这里用了 JWT ，这里用了 redis

问题是，你写的这些都是主流的选择，除非特别新特别小众的技术栈，否则在面试官看来就是能力平平

你就算是写技术上的细节，你也写的细一点
例如同样是用 JWT ，写得好就可以写成
“自定义 Spring Security 相关接口，使用 redis 和 token ，结合拦截器实现用户认证、token 刷新和权限认证机制”

你要这么写，我就信你是敲过代码的，否则这些玩意，哪个脚手架没有

你这种人事系统，一抓一大把
你可以写你实现的功能点上去，起码让面试官知道这个系统开发了什么功能，你有思考过哪些业务场景

从我的角度来看，应届生我不期待他有多少能力，八股文我也不问
但是实际敲过代码很重要，会额外的技术栈也很加分（例如小程序、有实际的云服务对接经历）
