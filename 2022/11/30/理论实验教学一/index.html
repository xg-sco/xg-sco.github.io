Linux 系统下的“Hello World”C 语言编程

环境准备, 使用的 linux 是版本是 centos7.9, 远程连接用的 XShell 工具
安装 gcc
首先要确保 centos 里面安装了 gcc,
安装 gcc 在终端输入 yum install gcc, 然后按 y 进行下一步就行
安装完成之后, 在终端输入 gcc -v 可以查看 gcc 的版本号

编写完成之后记得按 Esc, 输入:wq 保存退出

gcc 动态编译 test.c 源程序
默认 gcc 直接编译使用的编译方式就是使用的动态编译的
使用 gcc test.c -o testShare 编译 test.c 文件, testShare 是编译之后的可执行文件, 使用./testShare 就可以运行可执行文件

静态编译 test.c 源程序
输入 yum install glibc-static 安装静态库, 只有安装了才能进行静态编译

输入 rpm -qa | grep glibc-static 可以查看是否安装完成成功

输入 gcc test.c -o testStatic -static 进行静态编译, 编译之后的可执行文件是 testStatic, ./testStatic 可以运行可执行文件

输入 ll, 发现 testShare 使用空间为 8360 位, testStatic 占用空间是 861288 位,
很明显 testStatic 比 testShare 的文件大很多
这也就是说静态编译出来的可执行文件比动态编译出来的可执行文件要大

总结: 1.静态编译：编译器在编译可执行文件时，把需要用到的对应动态链接库(.so 或.ilb)中的部分提取出来，链接到可执行文件中去， 使可执行文件在运行时不需要依赖于动态链接库. 2.动态编译: 动态编译的可执行文件需要附带一个的动态链接库，在执行时，需要调用其对应动态链接库中的命令。所以其优点一
方面是缩小了执行文件本身的体积，另一方面是加快了编译速度，节省了系统资源。缺点一是哪怕是很简单的程序，只用到了链接库中的一两条命令，也需要附带一个相对庞大的链接库；二是如果其他计算机上没有安装对应的运行库，则用动态编译的可执行文件就不能运行。
